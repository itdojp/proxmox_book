## 3. 構成上の気になる点（候補）
- 章末の「まとめ（要点整理）」が明示されていない章が多い  
  - 第2章/第10章は「まとめ」相当の節がある一方で、第1章・第3章〜第9章は「よくあるつまずきポイント」や結びの段落で終わっており、読者が「この章で持ち帰るべき要点」と「次に読む章」を判断しにくい。
- 章構成テンプレートが完全には揃っていない  
  - 多くの章は `章のゴール` / `分かること・分からないこと` / `用語メモ` / `チェックリスト` といった骨格があるが、Part 0（環境準備）には `章のゴール` がない、など例外がある。
- 章内に「編集・制作メモ（TODO/Issue 参照）」が本文として混在している  
  - 第3章・第4章のスクリーンショット取得メモ（Issue #2 参照など）は、読者向け本文としては流れが途切れやすい。
- 「VM とコンテナ」の扱いが章構成としては曖昧になり得る  
  - 第1章・第2章では LXC を含む説明がある一方、Part I 以降の「手を動かす」対象は VM 中心になっているため、初学者が「コンテナも同じ粒度で扱うのか」を誤解しやすい。
- 第2章の `章のゴール` と `本章のねらい` の役割が重複している可能性  
  - どちらも「この章で何を狙うか」の説明になっており、テンプレートとしての一貫性（どの章にも同じ要素があるか）が揺らぐ。

## 4. 構成改善の提案（案）
- 章末に「まとめ（要点）+ 次章への接続」を追加し、章単位の学習完了を明確化する  
  - 対象: 第1章・第3章〜第9章（第2章・第10章は既に「まとめ」相当あり）  
  - 例: `まとめ` → `次に読む章（推奨）`（または `次章で扱うこと`）の2節構成にする。
- 章構成テンプレートを明文化し、各章に適用する（文書化のみでも可）  
  - 例: `章のゴール` → `分かる/分からない` → `用語メモ（最小）` → `事前準備（任意）` → `本論` → `よくあるつまずき` → `まとめ`。  
  - Part 0（環境準備）も同テンプレートの適用可否を検討し、適用する場合は `章のゴール` を追加する。
- 「編集・制作メモ」を本文から分離する（読者向け本文の連続性を優先）  
  - 例: スクリーンショット取得対象の一覧は `project-management/assets.md` 等に集約し、本文側は「該当スクリーンショットは `images/...` を参照」のように参照だけ残す。
- LXC（コンテナ）の扱いを構成として明確化する  
  - 方向性A: 本書は VM 中心である旨を第1章・第2章・第4章あたりで明示し、LXC は概念紹介に留める。  
  - 方向性B: Part I に「LXC コンテナの基本操作」節（または章）を追加し、VM と同程度の入門体験を用意する。  
  - いずれも「第1章で触れる内容」と「後続章で実施する内容」を対応づける目的。
- 第2章の `本章のねらい` の位置づけを整理する  
  - `章のゴール` と統合する（冗長性を下げる）か、あるいは `本章のねらい` を「本章の読み方（初心者向け）」等に置換して役割を明確にする。
